# 贡献指南（CONTRIBUTING）

感谢你愿意为 KeepAliveTabs Kit 做贡献。

## 目标与范围

本项目的定位是“最小闭环 Demo + 可复用方案参考”。因此 PR 的优先级按以下顺序排序：

1. 修复会影响主流程验证的问题（多页签、缓存、刷新恢复、错误兜底）
2. 降低依赖、减少复杂度、提升可读性
3. 增强文档（更清晰的使用/限制/常见坑说明）

不接受的变更（除非讨论后确有必要）：

- 引入大型依赖以实现“非核心”功能（例如请求库、状态管理、拖拽排序、复杂 UI 框架）
- 将 Demo 扩展为“业务脚手架”

## 开发与验证

安装依赖：

```bash
pnpm install
```

本地开发：

```bash
pnpm dev
```

类型检查：

```bash
pnpm typecheck
```

构建：

```bash
pnpm build
```

## 提交规范

- 提交信息使用简短、明确的中文或英文均可
- 变更尽量保持小步、可回滚
- PR 描述中写清楚：问题、原因、解决方案、验证方式

## Issue

提交 Issue 时建议提供：

- 复现步骤（从 `pnpm dev` 开始）
- 实际结果与期望结果
- 浏览器版本、Node/pnpm 版本
- 相关截图或控制台错误信息
